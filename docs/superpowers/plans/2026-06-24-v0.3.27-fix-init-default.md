**Goal:** Fix the v0.3.26 regression where `wiki init` defaults to `wiki/` subdir while the read path defaults to flat layout (`pages_dir = ""`). Align scaffold with read path, add `--subdir` escape hatch, and address v0.3.26 self-critic findings.

4. Add missing `embed` exclusion integration test
5. Add config key validation in `wiki config validate`
6. Update docs to reflect new defaults

/// Validate a config file against known keys. Returns warnings for
/// unknown keys under `[wiki]` and `[nim]` sections. Used by
/// `wiki config validate` to catch typos.
pub fn validate_config_file(path: &Path) -> Result<Vec<String>, WikiError> {
    if !path.exists() {
        return Ok(vec![]);
    }
    let text = std::fs::read_to_string(path)?;
    let value: toml::Value = text.parse().map_err(|e| WikiError::ConfigInvalid {
        path: path.display().to_string(),
        line: 0,
        message: format!("TOML parse error: {e}"),
    })?;
    let mut warnings = Vec::new();
    if let Some(table) = value.as_table() {
        if let Some(wiki) = table.get("wiki").and_then(|v| v.as_table()) {
            for key in wiki.keys() {
                if !KNOWN_WIKI_KEYS.contains(&key.as_str()) {
                    warnings.push(format!("unknown key in [wiki]: {key}"));
                }
            }
        }
        if let Some(nim) = table.get("nim").and_then(|v| v.as_table()) {
            for key in nim.keys() {
                if !KNOWN_NIM_KEYS.contains(&key.as_str()) {
                    warnings.push(format!("unknown key in [nim]: {key}"));
                }
            }
        }
    }
    Ok(warnings)
}
```

## v0.3.27 — UX Polish

**Bugfix — `wiki init` default aligned with read path:**
- Plain `wiki init` now scaffolds **flat layout** (pages at workspace root) by default,
  matching the `pages_dir = ""` read-path default introduced in v0.3.26. New users
  see their pages immediately after `wiki init` without editing config.
- `wiki init --subdir` explicitly opts into the legacy `wiki/` subdirectory layout.
  `--flat` is still accepted for backward compatibility with v0.3.26 scripts.
  `--flat` and `--subdir` are mutually exclusive.

**Bugfix — `wiki.exclude_dirs` is now additive:**
- User-provided `exclude_dirs` now **merges** with the built-in defaults
  (node_modules, .git, .opencode, etc.) instead of replacing them. This
  prevents users who add a single custom exclude from accidentally losing
  the entire default exclusion list. The CHANGELOG notes this as a
  breaking change from v0.3.26's replacement semantics.

**New feature — config key validation:**
- `wiki config validate` now warns on unknown keys under `[wiki]` and `[nim]`
  sections. Typos like `pages_dirr = "wiki"` or `embed_modle = "..."` are
  flagged instead of silently ignored.

**New test — `embed` exclusion:**
- Integration test verifies `wiki embed` skips pages under excluded directories.
```

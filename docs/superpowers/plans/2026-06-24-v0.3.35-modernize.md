# v0.3.35 Modernization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the dynamic `serde_yaml::Value` frontmatter with a typed `Frontmatter` struct, migrate `serde_yaml` → `serde-saphyr`, add a selective `clippy.toml` and a `deny.toml` policy file — all in a single v0.3.35 release.

**Architecture:** Same SSOT pattern as v0.3.25 (`build.rs` `include!`s the typed struct; the same struct generates the JSON Schema, parses YAML at runtime, and powers lint checks). One new struct file (`src/core/frontmatter.rs`), one new schema file (`skills/references/frontmatter.schema.json`), two new repo-root config files (`clippy.toml`, `deny.toml`), plus dep swap and downstream-consumer updates.

**Tech Stack:** Rust 2021, MSRV 1.88, serde 1, schemars 1.0, serde-saphyr (replacing serde_yaml 0.9), cargo-deny (local-only, not CI-gated in this release).

```rust
use std::collections::BTreeMap;

/// Typed representation of a wiki page's YAML frontmatter.
///
/// The 21 known fields are the keys with clear semantic purpose
/// that appear in at least one of the 4 real flat-layout wikis
/// (`mevin`, `minimax-code-wiki`, `MyWiki`, `PharmaWiki`). 15 of
/// 21 meet an original ≥50-occurrence target; 6 fall below but are
/// kept because each captures a distinct semantic category. See
/// `minimax-code-wiki`, `MyWiki`, `PharmaWiki`). See
/// `docs/superpowers/specs/2026-06-24-frontmatter-field-audit.md`.
///
/// `extra` captures all other keys (per-wiki taxonomy extensions like
/// `avatar`, `timezone`, `license`, `requiresBeta`, `session`,
/// `schedule`, `retired`, `triggers`, etc.) so they are preserved
/// without forcing a closed schema.
#[derive(Debug, Clone, serde::Deserialize, serde::Serialize, schemars::JsonSchema)]
pub struct Frontmatter {
    pub title: Option<String>,
    #[serde(default)]
    pub tags: Vec<String>,
    #[serde(rename = "type")]
    pub page_type: Option<String>,
    #[serde(default)]
    pub sources: Vec<String>,
    pub confidence: Option<f64>,
    pub created: Option<String>,
    pub updated: Option<String>,
    pub schema_version: Option<i64>,
    pub status: Option<String>,
    pub kind: Option<String>,
    pub domain: Option<String>,
    pub maturity: Option<String>,
    pub reviewed: Option<String>,
    #[serde(default)]
    pub aliases: Vec<String>,
    pub description: Option<String>,
    #[serde(default)]
    pub related: Vec<String>,
    pub source_type: Option<String>,
    pub sha256: Option<String>,
    pub ingested: Option<String>,
    pub name: Option<String>,
    #[serde(default)]
    pub descriptions: Vec<String>,
    #[serde(flatten)]
    pub extra: BTreeMap<String, serde_json::Value>,
}

impl Default for Frontmatter {
    fn default() -> Self {
        Self {
            title: None,
            tags: Vec::new(),
            page_type: None,
            sources: Vec::new(),
            confidence: None,
            created: None,
            updated: None,
            schema_version: None,
            status: None,
            kind: None,
            domain: None,
            maturity: None,
            reviewed: None,
            aliases: Vec::new(),
            description: None,
            related: Vec::new(),
            source_type: None,
            sha256: None,
            ingested: None,
            name: None,
            descriptions: Vec::new(),
            extra: BTreeMap::new(),
        }
    }
}
```

```rust
// Include the canonical type definitions and their `default_*` helpers.
// These files are also included by `src/core/config.rs` and
// `src/cli/doctor.rs` for the runtime types. This is the single source
// of truth for both runtime serialization and schema generation —
// adding a field to Config, DoctorReport, or Frontmatter automatically
// updates the generated JSON Schema. The default_* helpers MUST live
// in the included type file (not in build.rs separately) because
// schemars 1.0 resolves them at macro-expansion time AND calls them at
// schema-gen time to populate the JSON Schema's `default` keyword entries.
include!("src/core/config_types.rs");
include!("src/cli/doctor_report.rs");
include!("src/core/frontmatter.rs");

use std::path::Path;

fn main() {
    println!("cargo:rerun-if-changed=build.rs");
    println!("cargo:rerun-if-changed=src/core/config.rs");
    println!("cargo:rerun-if-changed=src/core/config_types.rs");
    println!("cargo:rerun-if-changed=src/cli/doctor.rs");
    println!("cargo:rerun-if-changed=src/cli/doctor_report.rs");
    println!("cargo:rerun-if-changed=src/core/frontmatter.rs");
    // `rust-embed` (in `src/skills/mod.rs`) emits its own
    // `cargo:rerun-if-changed=` lines for every file in
    // `src/skills/data/`, so we don't need to list them here.

    emit_json_schema::<Frontmatter>(
        "skills/references/frontmatter.schema.json",
        schemars::schema_for!(Frontmatter),
    );
}

fn emit_json_schema<T: schemars::JsonSchema>(path: &str, schema: schemars::Schema) {
    let json = serde_json::to_string_pretty(&schema).expect("serialize schema");
    let dest = Path::new(path);
    if let Some(parent) = dest.parent() {
        std::fs::create_dir_all(parent).expect("create schema parent dir");
    }
    std::fs::write(dest, format!("{json}\n")).expect("write schema");
}
```

```rust
use crate::core::frontmatter::Frontmatter;
use crate::error::WikiError;
use regex::Regex;
use std::sync::LazyLock;

#[derive(Debug, Clone)]
pub struct ParsedPage {
    pub frontmatter: Frontmatter,
    pub body: String,
}

pub fn parse_frontmatter(content: &str) -> Result<ParsedPage, WikiError> {
    let trimmed = content.trim_start_matches('\u{feff}'); // strip BOM
    if !trimmed.starts_with("---") {
        return Ok(ParsedPage {
            frontmatter: Frontmatter::default(),
            body: content.to_string(),
        });
    }
    // Find closing ---
    let after_first = &trimmed[3..];
    let after_first = after_first.strip_prefix('\n').unwrap_or(after_first);
    let end = after_first
        .find("\n---")
        .ok_or_else(|| WikiError::Other(anyhow::anyhow!("unclosed frontmatter")))?;
    let yaml_text = &after_first[..end];
    let body_start = end + 4; // skip \n---
    let body = after_first[body_start..]
        .strip_prefix('\n')
        .unwrap_or(&after_first[body_start..])
        .to_string();

    let frontmatter: Frontmatter = if yaml_text.trim().is_empty() {
        Frontmatter::default()
    } else {
        serde_yaml::from_str(yaml_text)?
    };
    Ok(ParsedPage { frontmatter, body })
}

// Keep the existing WIKILINK_RE, FOOTNOTE_DEF_RE, FOOTNOTE_REF_RE statics and the
// extract_wikilinks / extract_footnotes / extract_footnote_refs functions unchanged.
```

```rust
use llmwiki_cli::core::markdown::parse_frontmatter;

#[test]
fn parse_frontmatter_returns_typed_struct() {
    let md = "---\ntitle: Hello\ntags: [a, b]\n---\nBody";
    let p = parse_frontmatter(md).unwrap();
    assert_eq!(p.frontmatter.title.as_deref(), Some("Hello"));
    assert_eq!(p.frontmatter.tags, vec!["a", "b"]);
    assert_eq!(p.body, "Body");
}

#[test]
fn parse_frontmatter_catches_known_field_typo_via_extra() {
    // "titel" (typo) is NOT a known field, so it lands in `extra`, not in `title`.
    let md = "---\ntitel: Hello\n---\n";
    let p = parse_frontmatter(md).unwrap();
    assert_eq!(p.frontmatter.title, None);
    assert_eq!(
        p.frontmatter.extra.get("titel").and_then(|v| v.as_str()),
        Some("Hello"),
    );
}

#[test]
fn parse_frontmatter_preserves_niche_extra_fields() {
    // avatar and timezone are niche fields used by some wikis but not all.
    let md = "---\ntitle: X\navatar: /img/x.png\ntimezone: Europe/Paris\n---\n";
    let p = parse_frontmatter(md).unwrap();
    assert_eq!(p.frontmatter.title.as_deref(), Some("X"));
    assert_eq!(p.frontmatter.extra.len(), 2);
    assert!(p.frontmatter.extra.contains_key("avatar"));
    assert!(p.frontmatter.extra.contains_key("timezone"));
}

#[test]
fn parse_frontmatter_handles_duplicate_top_level_keys() {
    // Duplicate keys must still error (both serde_yaml and serde-saphyr reject).
    let md = "---\ntype: page\ntype: query\n---\n";
    assert!(parse_frontmatter(md).is_err());
}

#[test]
fn parse_frontmatter_handles_empty_yaml() {
    let md = "---\n---\nBody after empty frontmatter";
    let p = parse_frontmatter(md).unwrap();
    assert_eq!(p.frontmatter.title, None);
    assert!(p.frontmatter.tags.is_empty());
    assert!(p.frontmatter.extra.is_empty());
    assert_eq!(p.body, "Body after empty frontmatter");

#[test]
fn parse_frontmatter_handles_pipeline_type_expression() {

    let md = "---\ntype: entity | concept | comparison | query | schema | summary\n---\n";

#[test]
fn parse_frontmatter_handles_no_frontmatter() {
    let md = "Just body content, no frontmatter at all.";
    let p = parse_frontmatter(md).unwrap();
    assert_eq!(p.frontmatter.title, None);
    assert!(p.frontmatter.extra.is_empty());
    assert_eq!(p.body, md);
```

**Files:**

```toml
# Selective clippy lints at warn level. Catches the killer bugs
# (UTF-8 boundary panics, swallowed errors, panic macros) without
# -D-warnings churn. Use #[expect(LINT_NAME, reason = "...")] for
# justified exceptions (Rust 1.81+).
#
# Reference: https://emschwartz.me/your-clippy-config-should-be-stricter/

# Don't Panic — UTF-8 boundary + array/string indexing panics
string_slice            = "warn"
indexing_slicing        = "warn"

# Don't Panic — Result/Option unwraps and panic! macros
unwrap_used             = "warn"
panic                   = "warn"

# Skipped (not enabled in v0.3.35):
#   todo / unimplemented / unreachable — would flag LazyLock init sites; defer
#   get_unwrap / panic_in_result_fn — value-add marginal; defer
#   let_underscore_future / let_underscore_must_use — no current async bugs;
#     defer until tokio usage grows
#   cast_possible_truncation / cast_possible_wrap / cast_precision_loss —
#     no current numeric bugs; defer
#   arithmetic_side_effects — Evan Schwartz (Apr 2026) found 85% noise; defer
```


- [ ] **Step 3: For each new warning, fix the code or add `#[expect(...)]`**

For each unique warning, decide:

**A. Fix the underlying code** (preferred where the fix is local and improves safety):
- For UTF-8 boundary risks (`string_slice`): use `.chars().take(n).collect::<String>()` or `.get(..n)` instead of `&s[..n]`.
- For `unwrap_used` on test assertions: convert `.unwrap()` to `.expect("reason")` where the reason is meaningful, or accept the lint with an annotation.
- For `panic` macros in non-test code: replace with `Result<_, WikiError>` returns.

**B. Add `#[expect(LINT_NAME, reason = "...")]`** where the warning is intentional and the code is correct:
- For LazyLock init sites: `#[expect(unwrap_used, reason = "LazyLock init; pattern is hardcoded and tested at first use")]`.
- For test-only assertions: `#[expect(unwrap_used, reason = "test assertion; failure is the test outcome")]` — but only if the test is clearer with `.unwrap()` than with `.expect("...")`.

**Hard rules:**
- Every `#[expect(...)]` MUST have a `reason = "...">` clause. No silent suppressions.
- Never `#[allow(...)]` (it has no audit trail; `#[expect]` is the modern alternative since Rust 1.81).
- If the same warning fires in > 5 places and the fix is mechanical, fix the code, don't add 5 `#[expect]` annotations.

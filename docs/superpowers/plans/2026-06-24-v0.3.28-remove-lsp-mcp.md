# v0.3.28 Implementation Plan — Remove LSP/MCP Servers

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fully remove `llmwiki-cli lsp` and `llmwiki-cli mcp` and replace their role with a `CONFIG` sub-skill + an enhanced hub workflow that teaches agents to use `wiki config show-effective`, `wiki config show-schema`, `wiki config validate`, and `wiki doctor`.

**Architecture:**
- Drop 3 source files (`src/cli/lsp.rs`, `src/cli/mcp.rs`, `src/core/lsp_domain.rs`) and 4 test files.
- Drop 2 Cargo dependencies (`tower-lsp-server`, `rmcp`) and 2 CLI subcommands (`Command::Lsp`, `Command::Mcp`).
- Drop 2 skill bundles (`marketplace/skills/wiki/LSP/`, `marketplace/skills/wiki/MCP/`).
- Add 1 sub-skill (`CONFIG/`) and a hub workflow section.
- Add `--section` filter to `wiki config show-schema` so agents can scope the schema dump.
- Bump version to `0.3.28`; CHANGELOG documents removal + migration.

**Tech Stack:** Rust 2021, MSRV `1.88`, `clap`, `serde`, `tokio`, `schemars`. No new deps; 2 deps removed.

```rust
/// Print the JSON Schema for the `Config` type. Optional `--section <wiki|nim>`
/// filters output to just that section so agents can scope schema discovery.
async fn cmd_show_schema(section: Option<String>) -> Result<(), WikiError> {
    let mut schema = schemars::schema_for!(crate::core::config::Config);
    if let Some(sec) = section.as_deref() {
        let key = match sec {
            "wiki" => "wiki",
            "nim" => "nim",
            _ => unreachable!("clap value_parser rejects this"),
        };
        // Drop the other section from the top-level properties.
        if let Some(props) = schema.as_object_mut().and_then(|o| o.get_mut("properties"))
            .and_then(|p| p.as_object_mut())
        {
            let drop: Vec<String> = props
                .keys()
                .filter(|k| k.as_str() != key)
                .cloned()
                .collect();
            for k in drop {
                props.remove(&k);
            }
        }
        // Also drop from `required`.
        if let Some(req) = schema.as_object_mut().and_then(|o| o.get_mut("required"))
            .and_then(|r| r.as_array_mut())
        {
            req.retain(|v| v.as_str() != Some(key));
        }
    }
    println!(
        "{}",
        serde_json::to_string_pretty(&schema).expect("schema serialization is infallible")
    );
    Ok(())
}
```

### Task 2: Add validate-after-edit integration test

**Files:**

- [ ] **Step 1: Write the test**

Append to `tests/config_cli_test.rs`:

```rust
#[test]
fn config_validate_after_edit_catches_typo() {
    // v0.3.28+: agents editing wiki-root.toml should run `wiki config validate`
    // after every change. This test simulates that workflow: write a valid
    // config, then write a typo'd config, and verify validate catches it.
    let tmp = tempfile::tempdir().unwrap();
    let reg_dir = tmp.path().join(".agents");
    std::fs::create_dir_all(&reg_dir).unwrap();
    let alias_path = tmp.path().to_path_buf();
    std::fs::write(
        reg_dir.join("wiki-root.toml"),
        format!(
            "[defaults]\npages_dir = \"\"\n\n[mevin]\npath = {}\n",
            serde_json::to_string(&alias_path.to_string_lossy()).unwrap_or_else(|_| format!("{:?}", alias_path))
        ),
    )
    .unwrap();

    // 1. Valid config — should pass.
    Command::cargo_bin("llmwiki-cli")
        .unwrap()
        .arg("--wiki").arg("mevin")
        .arg("config").arg("validate")
        .assert().success();

    // 2. Introduce a typo — should still succeed but warn on stderr.
    std::fs::write(
        reg_dir.join("wiki-root.toml"),
        format!(
            "[defaults]\npages_dur = \"\"\n\n[mevin]\npath = {}\n",
            serde_json::to_string(&alias_path.to_string_lossy()).unwrap_or_else(|_| format!("{:?}", alias_path))
        ),
    )
    .unwrap();

    let output = Command::cargo_bin("llmwiki-cli")
        .unwrap()
        .arg("--wiki").arg("mevin")
        .arg("config").arg("validate")
        .assert()
        .success()
        .get_output().clone();
    let stderr = String::from_utf8_lossy(&output.stderr);
    assert!(
        stderr.contains("unknown key in [defaults]: pages_dur") || stderr.contains("✓"),
        "validate must surface unknown-key warning: {stderr}"
    );
}
```

```markdown
- The LSP server (`llmwiki-cli lsp`) and MCP server (`llmwiki-cli mcp`)
  are removed. The CLI is now a single-purpose file tool.
- Skills `lsp/` and `mcp/` are removed from the marketplace bundle.
- Source files removed: `src/cli/lsp.rs`, `src/cli/mcp.rs`,
  `src/core/lsp_domain.rs` (shared domain, only used by the servers).
- Dependencies removed: `tower-lsp-server`, `rmcp`.
- 4 protocol-level tests removed (`lsp_*`, `mcp_*`).
- ~1,475 lines removed across source + skills + tests + docs.

**Migration:** if you wired `llmwiki-cli lsp` into your editor (Helix,
Neovim, Zed, VS Code) or `llmwiki-cli mcp` into your AI host (Claude
Desktop, Claude Code, Cursor, Codex), remove the config block — the
subcommands no longer exist.

**Added — `wiki config show-schema --section <wiki|nim>`:**
- Filter the schema output by section. Agents writing configs no
  longer need to scroll through 200+ lines of JSON.

**Added — `CONFIG` sub-skill:**
- Teaches AI agents the edit-config workflow:
  show-effective → show-schema → edit → validate → doctor.
- Replaces the role the `lsp/` and `mcp/` skills filled.
```

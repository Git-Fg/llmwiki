# Wiki Skill (Full Guide)

> This is the full skill content for the `wiki` knowledge base. The stub generated by `build.rs` points at this content via `wiki skill show`. Load topics on demand with `wiki skill show <topic>`.

## What This Is

A markdown-based personal knowledge base, following Andrej Karpathy's LLM Wiki pattern. The wiki is plain Markdown files in a git repo. Compiled pages live in `wiki/`, source materials in `raw/`, embeddings in `embeddings.jsonl` (gitignored). No database.

## When to Use

Trigger this skill when the user:
- Asks to ingest a file, article, paper, URL, or note
- Asks a question that prior research might answer
- Asks to lint, maintain, or sync the wiki
- Asks to set up the wiki on a new device
- References the wiki by name or path

## File Layout (User's Wiki)

```
~/my-wiki/
├── .wiki/config.yaml      # workspace config (committed)
├── wiki/                  # compiled markdown (committed)
├── raw/                   # source materials (committed)
├── index.md               # catalog (committed)
├── log.md                 # operational log (committed)
└── embeddings.jsonl       # NIM vectors (GITIGNORED, per-device)
```

## Conventions

- Filenames: lowercase-hyphen, regex `^[a-z0-9]+(?:-[a-z0-9]+)*\.md$`
- Every page starts with YAML frontmatter
- Required fields: `title`, `created`, `updated`, `tags`, `sources`
- Optional: `type` (entity/concept/comparison/query/summary), `confidence`, `contested`, `review_by`
- Wikilinks: `[[page-name]]`, minimum 2 outbound per page
- Add new pages to `index.md` alphabetically
- Append structured entries to `log.md`: `## [YYYY-MM-DD] action | description`

## Tooling Preference

1. **Read / Grep / Glob** for browse and exact match
2. **`wiki search <query>`** for conceptual/semantic lookup
3. **`wiki query <question>`** for synthesized RAG answers
4. **`wiki lint`** before commits and after broad edits
5. **`wiki status`** to check wiki state

## Core Workflows

### Ingest

1. Place source in `raw/articles/` (or appropriate `raw/<subdir>/`)
2. `wiki ingest <path>` — computes sha256, adds frontmatter, compiles to wiki pages
3. Verify pages look right, then `git add . && git commit -m "ingest: X"`

### Query

1. Read `index.md` first for routing
2. For exact match: `Read` the page
3. For semantic: `wiki search <query>`
4. For synthesized answer: `wiki query <question>`
5. If answer is substantive: file as new page in `wiki/`

### Lint

1. `wiki lint` — exit 0 = clean, exit 2 = errors
2. `wiki lint --fix` to auto-fix safe issues
3. Always lint before commits

### Sync

1. `git pull` — get latest markdown
2. `wiki embed --skip-existing` — embed any new pages locally
3. Edit / commit / push
